ABAP CDS - DEFINE ROLE, grant_rule

Syntax

GRANT SELECT ON cds_entity;

Effect

Full control rule in statement DEFINE ROLE of the CDS DCL. A full control rule GRANT SELECT ON without the addition WHERE provides access to a CDS entity cds_entity without conditions. If cds_entity is specified, the same applies as for a conditional access rule.

Notes

  • A full access rule has the same effect for a CDS entity as when there is no role for the CDS entity or when the CDS access control for this CDS entity is suppressed by the annotation @AccessControl.authorizationCheck with the value #NOT_ALLOWED.
  • Usually SAP does not supply any CDS roles with full access rules. Partners and customers can use full access rules to override roles supplied by SAP.
  • The use of a full access rule leads to a warning being raised by the syntax check.

Example

The following CDS role means that all users can access the CDS view DEMO_CDS_AUTH_FULLACCESS without access conditions:

@MappingRole: true
define role demo_cds_role_fullaccess {
   grant select on demo_cds_auth_fullaccess; }
