Logical Database Schemas in SAP HANA

A logical database schema is a symbolic name for a database schema in the SAP HANA database.

  • Definition
  • Mapping Physical Database Schemas
  • Use

Definition

A logical database schema is a transportable repository object that developers must create in the ABAP Development Tools (ADT). The only additional attributes in the repository object are its name, which enables its use and to which a physical database schema can be mapped.

Note

Developers with development authorization can create and modify a logical database schema.

Mapping Physical Database Schemas

A physical database schema is mapped to an existing logical database schema using the documented administration transaction DB_SCHEMA_MAP. A mapping of this type is currently only possible on an AS ABAP whose central database is a SAP HANA database. The physical database schema is case-sensitive.

A mapping made using the transaction DB_SCHEMA_MAP is usually a local mapping, but can be transported with some restrictions. Transports like this are usually only made from systems with an SAP HANA database and should only be made into systems in which the specified physical database schemas exist. Transports should only be made between systems in the same enterprise. Any mappings made by SAP are not delivered to customers. The cross-client database table AMDP_SCHEMA_MAP, in which the mapping is saved, has the delivery class C.

The mappings exist in systems that do not have an SAP HANA database as a central database, but cannot be edited and are ignored.

Note

System administrators with appropriate authorizations can map physical database schemas.

Use

A logical database schema can be used by frameworks for variable access to a physical database schema in Native SQL. More specifically, accesses like this are supported by the ABAP Managed Database Procedures (AMDP).
