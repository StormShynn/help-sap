byte order

Determines the order in which the binary representation of the data types i, int8, decfloat16, decfloat34, f, s, or a character is stored in the memory. A distinction is made between big and little endian. In the first case, the byte with the highest level, and in the second case the byte with the lowest level, is written to the first position in the memory. The byte order on the current application server can be derived from the static attribute ENDIAN in the system class CL_ABAP_CHAR_UTILITIES.
