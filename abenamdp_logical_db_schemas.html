AMDP - Logical Database Schemas

In an AMDP method, a full qualified name can be specified to access objects in a different database schema. Here, the AMDP framework supports the logical database schemas for the SAP HANA database. A logical database schema to which a physical name is mapped can be specified instead of the physical name of a database schema This is done using the following the syntax:

"$ABAP.schema( [name =] schema_name )"

The ABAP runtime environment replaces this expression (including the quotation marks in the implementation on the database) with the physical database schema mapped to the logical database schema using schema_name. The specified logical database schema is not case-sensitive, but the mapped physical database schema is.

The specified logical database schema must exist as a repository object. If a physical database schema is mapped to the logical database schema, the physical schema must exist on the current database. If no physical database schema is mapped to the logical database, a syntax check warning occurs. When the AMDP method is executed, an exception of the class CX_AMDP_DBPROC_GENERATE_FAILED is raised.

Notes

  • This syntax is currently only possible in the implementation language SQLScript.
  • When database objects in the current database schema are accessed, the name of this schema cannot be specified (either directly or using logical database schemas)
  • If a logical database schema is deleted or the mapping of a physical database schema is modified, the ABAP runtime environment checks all consumer AMDP classes.
