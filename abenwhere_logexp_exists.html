Quick Reference

sql_cond - EXISTS

Syntax

... EXISTS ( SELECT subquery_clauses [ UNION ...] ) ...

Effect

Checks the results set of a subquery. The expressions is true when the results set of the subquery defined by the clauses subquery_clauses contains at least one row. This expression is possible for all results sets that can be defined using the clauses subquery_clauses. The language element UNION can be used to combine the result sets of multiple subqueries. In this case, additional rules query_clauses apply for specifying clauses.

Example

Extracts all available flights leaving from New York from the database table SFLIGHT to the internal table free_flights.

DATA city         TYPE spfli-cityfrom VALUE 'NEW YORK'.

SELECT *
       FROM sflight AS s
       WHERE seatsocc < s~seatsmax AND
             EXISTS ( SELECT  *
                             FROM spfli
                             WHERE carrid   =  s~carrid AND
                                   connid   =  s~connid AND
                                   cityfrom = @city )
       INTO TABLE @DATA(free_flights).
