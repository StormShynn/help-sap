Assertion

Assertion in an ABAP program. An assertion is defined by an ASSERT statement as a conditional checkpoint. Assertions are either always active or can be activated by assigning them to a checkpoint group. When the program reaches an active assertion, it evaluates the corresponding condition. If the condition is violated, the program terminates with a runtime error, accesses the ABAP Debugger, or creates a log entry. If the assertion is assigned to a checkpoint group, the program behavior is controlled by the corresponding activation settings, and otherwise, the program terminates.
