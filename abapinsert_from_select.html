INSERT dbtab - subquery_clauses

Quick Reference

Syntax

... { FROM source
      FIELDS select_clause }
  | { select_clause
      FROM source }
      [WHERE sql_cond]
      [ GROUP BY group] [HAVING group_cond]
      [db_hints]  ...

Effect

Possible clauses and additions of a subquery after addition FROM of statement INSERT. These clauses define a result set, which is used as a data source of the INSERT statement.

With the exception of the client column, the results set of the subquery must have at least as many columns as the primary key of the database table or the classic view filled using the INSERT statement. It cannot, however, have more columns than the database table or view being filled.

The following special rules apply:

  • FROM clause
  • In the FROM clause, it is not possible to access the database table or classic view, which are filled using the INSERT statement.
  • In the FROM clause, additions USING CLIENT and CLIENT SPECIFIED cannot be used.
  • If the database table or classic view filled using the INSERT statement is client-specific, at least one data source of the subquery must be client-specific. The automatic client handling of the subquery is performed in accordance with the client handling of the INSERT statement. If specified, an addition USING CLIENT of the INSERT statement also applies to the subquery.
  • SELECT clause
  • Any SELECT lists of the subquery specified as comma-separated lists must define one column of the results set for at least every column of the database table or classic view being filled, with the exception of the client column of the primary key. No client column can be specified as the first column of the SELECT list. In other positions, client columns are not interpreted as client columns.
  • In a SELECT list of the subquery specified using *, client columns are ignored. The results set produced here must contain one column for at least every column of the primary key, with the exception of the client column of the database table or classic view being filled.
